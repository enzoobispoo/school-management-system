## O que multi-tenancy significa para esse sistema

**O problema central:** hoje o banco tem uma única "escola" (`EscolaSettings` com `id = "default"`). Todos os alunos, cursos, professores e pagamentos pertencem a esse único contexto. Se você colocar duas escolas no mesmo sistema, elas veriam os dados uma da outra.

## O que precisa mudar — em ordem

### 1. Schema (banco de dados)
Adicionar `schoolId` em **todos os models de dados**:

```
Aluno, Curso, Professor, Turma, TurmaHorario, Matricula,
Pagamento, CobrancaEnvio, CobrancaLote, Evento, Notificacao,
UserInvite, AiAuditLog, AiChatSession
```

E vincular `User` a uma `School`:
```prisma
model School {
  id        String   @id @default(cuid())
  nome      String
  createdAt DateTime @default(now())
  users     User[]
  // ... todos os outros models
}
```

### 2. Auth — JWT precisa carregar `schoolId`
Hoje o token tem `{ userId, email, role }`. Precisa virar `{ userId, email, role, schoolId }`. Toda query usa esse `schoolId` para filtrar.

### 3. Todas as ~63 rotas de API
Cada `prisma.aluno.findMany()` vira `prisma.aluno.findMany({ where: { schoolId } })`. Idem para todos os outros models.

### 4. Middleware (proxy.ts)
Verificar se o `schoolId` do token bate com o recurso sendo acessado.

### 5. Scripts de seed e admin
O `create-admin.mjs` precisa criar uma `School` primeiro e vincular o admin a ela.
